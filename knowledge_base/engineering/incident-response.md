# Incident Response

## Definition

An incident is any event that disrupts service availability.

## Steps

1. Identify issue
2. Notify team in Slack #incidents
3. Assign incident owner
4. Mitigate issue
5. Write postmortem

## Severity Levels

* SEV1: full outage
* SEV2: partial degradation
* SEV3: minor issue

## Postmortem

Must be completed within 48 hours
