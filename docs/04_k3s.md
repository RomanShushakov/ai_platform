# ☸️ K3s Cluster Setup (Raspberry + Jetson)

This guide sets up a **lightweight Kubernetes (K3s) cluster** across:

* 🧠 Raspberry Pi → control-plane
* ⚡ Jetson → worker node (GPU)

---

# 🎯 Goal

By the end of this guide, you will have:

* a working K3s cluster
* multi-node scheduling
* NFS accessible from pods
* a private container registry
* ability to deploy custom images

---

# 🧭 Setup Flow

```text
K3s Server → K3s Agent → Verify → Test Deployment → NFS → Registry → Custom Images
```

---

# 1️⃣ Install K3s (Server + Agent)

## ▶️ Install Server (Raspberry)

```bash
ansible-playbook k3s/playbooks/k3s_server.yml -K
```

---

## ⚠️ Enable cgroups (REQUIRED on Raspberry)

Edit:

```bash
sudo cp /boot/firmware/cmdline.txt /boot/firmware/cmdline.txt.bak
sudo vi /boot/firmware/cmdline.txt
```

Append to the **same single line**:

```text
cgroup_memory=1 cgroup_enable=memory
```

Reboot:

```bash
sudo reboot
```

Restart K3s:

```bash
sudo systemctl restart k3s
sudo systemctl status k3s
```

---

## ▶️ Install Agent (Jetson)

```bash
ansible-playbook k3s/playbooks/k3s_agent.yml -K
```

Verify:

```bash
ansible-playbook k3s/playbooks/k3s_verify.yml -K
```

---

## 🔍 Verify Cluster

On Raspberry:

```bash
kubectl get nodes -o wide
```

Expected:

* Raspberry = control-plane
* Jetson = worker

---

## 🔄 Rollback (if needed)

```bash
ansible-playbook k3s/playbooks/k3s_agent_rollback.yml -K
ansible-playbook k3s/playbooks/k3s_server_rollback.yml -K
```

---

# 2️⃣ Deploy Test Application

Deploy nginx:

```bash
kubectl create deployment hello-k3s --image=nginx:alpine
kubectl expose deployment hello-k3s --type=NodePort --port=80
```

Check:

```bash
kubectl get pods -o wide
kubectl get svc hello-k3s
```

Expected:

* pod scheduled (often on Jetson)
* service accessible via NodePort

---

# 3️⃣ Test NFS Inside Kubernetes

## ▶️ Create Test Pod

On Raspberry:

```bash
cat <<'EOF' > nfs-test-pod.yaml
apiVersion: v1
kind: Pod
metadata:
  name: nfs-test
spec:
  nodeSelector:
    kubernetes.io/hostname: jetson
  containers:
    - name: shell
      image: alpine:latest
      command: ["sleep", "3600"]
      volumeMounts:
        - name: nfs-data
          mountPath: /mnt/nfs
  volumes:
    - name: nfs-data
      hostPath:
        path: /home/roman/nfs
        type: Directory
EOF
```

Apply:

```bash
kubectl apply -f nfs-test-pod.yaml
kubectl get pod nfs-test -o wide
```

---

## ▶️ Verify Inside Pod

```bash
kubectl exec -it nfs-test -- sh
```

Inside pod:

```bash
ls -la /mnt/nfs
ls -la /mnt/nfs/slurm/job-examples
echo hello-from-k3s-pod > /mnt/nfs/k3s-test.txt
exit
```

---

## ▶️ Verify Outside

On Raspberry:

```bash
cat ~/nfs/k3s-test.txt
```

On Jetson:

```bash
cat ~/nfs/k3s-test.txt
```

---

## ▶️ Cleanup

```bash
kubectl delete pod nfs-test
kubectl delete service hello-k3s
kubectl delete deployment hello-k3s
```

---

# 4️⃣ Setup Private Docker Registry

Deploy registry on Jetson:

```bash
ansible-playbook k3s/playbooks/k3s_registry.yml -K
ansible-playbook k3s/playbooks/k3s_registries_config.yml -K
ansible-playbook k3s/playbooks/k3s_registry_verify.yml -K
```

---

## 🔄 Rollback

```bash
ansible-playbook k3s/playbooks/k3s_registry_rollback.yml -K
ansible-playbook k3s/playbooks/k3s_registries_config_rollback.yml -K
```

---

# 5️⃣ Push Image to Registry

## ⚠️ Important

Use **Jetson LAN IP**, not Tailscale IP:

```text
192.168.178.103:5000
```

---

## ▶️ Push Test Image

```bash
docker pull nginx:alpine
docker tag nginx:alpine 192.168.178.103:5000/nginx-test:latest
docker push 192.168.178.103:5000/nginx-test:latest
```

---

## 🧠 Note

For custom images (ARM64):

* use `docker buildx`
* build for `linux/arm64`
* push to registry

---

# 6️⃣ Deploy Image from Registry

```bash
kubectl create deployment registry-test \
  --image=192.168.178.103:5000/nginx-test:latest
```

Expose:

```bash
kubectl expose deployment registry-test \
  --type=NodePort \
  --port=80
```

Verify:

```bash
kubectl get pods -o wide
kubectl get svc registry-test
```

Expected:

* image pulled from private registry
* pod running on cluster
